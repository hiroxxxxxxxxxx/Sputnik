# **マクロインカム戦略 完全運用定義書（自動化対応版・2026年2月改訂）**

## **0\. 戦略概要**

### **0-1. システム憲章**

**Ⅰ. 目的** 本システムは、資本の長期的存続と複利成長を目的とする。 最大化は目指すが、破壊を伴う最大化は拒否する。

**Ⅱ. 設計原則** 本システムは状態遷移制御系である。 エンジン出力は3段階とする。 連続係数は最終的に離散状態へ収束する。 抽象レイヤーは混合しない。

**Ⅲ. 抽象構造** 市場はICL・SCLに抽象化される。 資本はLCLに抽象化される。 市場因子は直接資本を制御しない。 ICL / SCL / LCL → Effective Level → Engine この順序を破らない。

**Ⅳ. ヒステリシス原則** 状態は瞬間値で決定しない。 遷移には持続条件を必要とする。 復帰は進入より遅くてよい。

**Ⅴ. 美学原則** 不要な自由度を増やさない。 例外を作らない。 局所最適のために構造を壊さない。 強度はレイヤー内で解決する。

**Ⅵ. 実用原則** 暴落時に自己改造しない。 勝ち相場で構造を緩めない。 市場環境の変化はパラメータで吸収する。 レイヤーは固定資産である。

**Ⅶ. 禁則事項** 市場因子を直接LCLへ注入しない。 恐怖を理由に例外処理を追加しない。 係数の微調整で本質を誤魔化さない。

**Ⅷ. 階層分離原則** 各層は自分より上位の層の意図を知ってはならない。 因子は自身がどの制御レベルに使われるかを知らない。 フライトコントローラーはEffective Level以外の下位情報を参照しない。 層をまたいだ直接参照は構造汚染である。

### **0-3. 運用の基本原則**

本戦略は、コックピットを指揮系統の起点とし、フライトコントローラーが情報を統合して運用判断を機械的に決定する。

**運用判断の構造**

| 名称 | 役割 | 実装・ロジック |
| ----- | ----- | ----- |
| フライトプラン | 戦略的介入と環境認識 | 金利サイクルや重要イベントに基づき、パラメータの切り替えやリスク許容量（スロットル）の抑制を決定する |
| 操縦制御 | パイロットによる運用モード管理 | マニュアル／半自動／全自動の切り替えと承認フローを管理する |
| フライトコントローラー | 情報の統合と機械的執行 | 取得した情報を評価し、最悪のシグナルを即採用。生存を最優先し、運用の可否・規模を非情に決定する |

**変数の定義と統合プロセス**

フライトコントローラーは、市場環境と自己の体力の情報を3層で評価し、最も保守的な判断を優先する。

■ 個別制御（ICL）：個別銘柄の環境評価 $$ICL\_{NQ} \= \\max(P, V, C)$$ $$ICL\_{GC} \= \\max(P, V, R)$$ （$P$: 価格, $V$: ボラティリティ, $C$: 信用, $R$: 実質金利）

■ 同期制御（SCL）：銘柄間トレンド相関評価 $$SCL \= T\\text{（相関ロジック）}$$ （$T$: NQ/GC銘柄間トレンド同期）

■ 制限制御（LCL）：資本の安全限界評価 $$LCL \= \\max(U, S)$$ （$U$: 証拠金使用率, $S$: SPAN乖離率）

■ 最終命令 $$Effective \= \\max(ICL\_{NQ}, ICL\_{GC}, SCL, LCL)$$

**非対称性の徹底**

* 降格（悪化時）：異常を検知した瞬間、フライトコントローラーはノータイムで運用を縮退・停止させる  
* 昇格（改善時）：設定された「継続期間」や「一定以上の回復幅」を満たさない限り、上位判定への復帰を認めない

**因子体系一覧**

| 区分 | 制御レベル | 計器名 | 観測項目 | 因子名 |
| ----- | ----- | ----- | ----- | ----- |
| 外部/個別（NQ） | ICL | 姿勢指示器 | 価格・移動平均（NQ1\!） | Price（P） |
| 外部/個別（NQ） | ICL | 対気速度計 | ボラティリティ（VXN） | Volatility（V） |
| 外部/個別（NQ） | ICL | 昇降計（NQ用） | 信用スプレッド（HYG/LQD） | Credit（C） |
| 外部/個別（GC） | ICL | 姿勢指示器 | 価格・移動平均（GC1\!） | Price（P） |
| 外部/個別（GC） | ICL | 対気速度計 | ボラティリティ（GVZ） | Volatility（V） |
| 外部/個別（GC） | ICL | 昇降計（GC用） | 実質金利（TIP） | Real-Rate（R） |
| 外部/同期 | SCL | 旋回計 | 2銘柄同期トレンド | Trend（T） |
| 内部/制限 | LCL | Gメーター | 証拠金使用率（MM/NLV） | Utilization（U） |
| 内部/制限 | LCL | タコメーター | SPAN乖離率 | SPAN（S） |

## **1\. 航空機推進・制御システム（Aeronautics & Propulsion）**

**［概念］** 本システムは、機体本体と、それに取り付けられた3つのエンジンポッドで構成される。各ポッドは、推進力（先物）と制御面（オプション）が一体化した独立駆動ユニットである。

### **1-1. ユニット構成**

**［概念］** メインエンジンが機体の生存を担う中枢として機能し、姿勢制御エンジンが巡航の安定を支え、ブースターが追い風局面での加速を担う。リスク発生時はブースターから外側に向けて順次カットし、機体本体の生存を最優先する。

**［仕様］**

| ユニット名 | 推進（先物） | 制御（OP） | 役割の論理 |
| ----- | ----- | ----- | ----- |
| メインエンジン | Mini先物（50%） | 防衛装甲（PB/BPS/CC） | 機体屋台骨：墜落を阻止し生還を担う中枢。PB・BPS・CCをすべて装備 |
| 姿勢制御エンジン | Micro先物（25%） | 逆噴射（PB） | 主力エンジン：巡航の要。異常時は逆噴射（PB）で機体を急減速させる |
| ブースター | Micro先物（25%） | 加速フラップ（BPS） | 補助エンジン：加速専用。リスク発生時は即座に停止（エンジンカット）する |

### **1-2. 燃料と油圧（資産基盤）**

**［概念］** T-Billは航続距離を決める燃料タンクであり、現金バッファは機体を動かす油圧系統である。油圧が枯渇すると機体は動かなくなるため、$30k割れで燃料タンクから補給する。

**［仕様］**

すべての運用のエネルギー源となる純資産（NLV $1M）。

| 項目 | 形態 | 金額 | 役割 |
| ----- | ----- | ----- | ----- |
| 燃料タンク | T-Bill（1年物） | $850k | 航続距離：担保価値（約94%）維持。3ヶ月ラダー運用 |
| 油圧系統（現金） | 現金バッファ | $150k | 即時動作：値洗い（油圧）原資。$30k割れで燃料タンク（T-Bill）から補給 |

### **1-3. ユニット動力機構（設計図・Blueprint）**

**［概念］** 各エンジンは設計図（LayerBlueprint）に従って組み立てられる。設計図は起動時に読み込まれ、飛行中は変更できない。NQ用とGC用を同構造で対称配分することで、どちらかの銘柄が急落しても機体全体への衝撃を分散する。

**［仕様］**

各エンジンはNQ専用またはGC専用の1インスタンスであり、層ごとの設計図（LayerBlueprint）に基づいて3層（メイン・姿勢制御・ブースター）で構成される。設計図はTOML等で定義し、起動時に読み込んで実行中は変更しない（0-1-Ⅵ）。

* **メインエンジン層**：Mini先物（NQ/GC）＋ PB ＋ CC ＋ BPS。設計図はMain用Blueprint  
* **姿勢制御エンジン層**：Micro先物（MNQ/MGC）＋ PB。CC・BPSは装着不可（4-1, 4-2）。設計図はAttitude用Blueprint  
* **ブースター層**：Micro先物（MNQ/MGC）＋ BPS。PB・CCは装着不可。設計図はBooster用Blueprint  
* **構成**：層ごとに「先物 ＋ 当該層で装着する操縦翼面」をBlueprintの比率で保持する。メイン層のみがPB/CC/BPSをすべて持つ  
* **対称配分（1:1）**：NQ用エンジンとGC用エンジンを同構造で組み立て、NQ系列とGC系列のエクスポージャを常に等価に保つ  
* **最大出力上限**：NLV $1Mに対し、総エクスポージャ $2M（200%）を上限とする

### **1-4. スロットルモードとパージ・シーケンス**

**［概念］** フライトコントローラーが決定するEffective Levelに応じて、外側のエンジンから順にカットする。追い風（Effective=0）では全発稼働、気流の乱れ（Effective=1）でブースターをカット、重大な危機（Effective=2）では姿勢制御エンジンもパージして機体本体だけで生存を図る。

**［仕様］**

**Boost（200%）：全発エンジン稼働**

* 構成：メインエンジン＋姿勢制御エンジン＋ブースター  
* 状態：追い風を利用した最大加速。全リソースを推進に投下

**Cruise（150%）：ブースター・エンジンカット**

* 構成：メインエンジン＋姿勢制御エンジン  
* 状態：気流の乱れ（Effective=1）でブースター（Micro先物）をカット。逆噴射（PB）を備えた安定航行へ

**Emergency（100%）：姿勢制御エンジン・パージ＆逆噴射全開**

* 構成：メインエンジンのみ  
* 状態：重大な危機（Effective=2）。本体以外の先物を全パージ。防衛装甲（PB）で生存を図る

### **1-5. 出力の遮断と復帰**

**［概念］** 遮断は外側から、復帰はメインエンジンに近い側から。この非対称な順序が、資本を守りながら機会を逃さない設計の核心である。

**［仕様］**

**出力遮断（パージ）：** フライトコントローラーが「レッドライン」を検知した際、即座に外側から切り離す。

1. ブースターをエンジンカット（200% → 150%）  
2. 姿勢制御エンジンをパージ（150% → 100%）

**再点火（環境復帰）：** 「昇格条件」を満たした場合にのみ、メインエンジンに近いユニットから順に再点火を許可する。

1. 姿勢制御エンジンを再点火（100% → 150%：Cruise復帰）  
2. ブースターを再点火（150% → 200%：Boost復帰）

## **2\. コックピット**

### **2-1. フライトプラン**

**［概念］** 本戦略では金利サイクルを「巡航高度」、運用方針を「ルート」として定義する。高度はFOMCの管制指示に従い設定し、一度設定したら6ヶ月間維持する。高度の変更は操縦翼面のパラメータのみに作用し、フライトコントローラーのロジックは高度によらず共通である。ただしV因子・C因子・R因子の閾値テーブル切り替えを例外とし、高高度・中高度と低高度で異なる定規を適用する。

**［仕様］**

月次でドットチャート・FF金利見通し・重要イベントを確認し、高度・ルートを設定する。

| 項目 | 高高度（利下げサイクル） | 中高度（安定期） | 低高度（利上げサイクル） |
| ----- | ----- | ----- | ----- |
| スロットル上限 | Boost（200%） | Boost（200%） | Cruise（150%） |
| 基本方針 | 上昇追従。先物成長＋BPSインカム | バランス型。時間価値回収＋踏み上げ警戒 | 防御優先。証拠金温存＋確実なインカム |
| CC方針 | なし（高高度では廃止） | メインエンジン稼働。P0時停止 | セータ重視。確実なプレミアム回収 |
| BPS方針 | OTM浅め（▲5〜7%） | OTM中間（▲7〜8%） | OTM深め（▲8〜10%） |
| PB方針 | 感度高め（Δ0.12） | 中間（Δ0.10） | コスト抑制（Δ0.08） |
| 主な収益源 | 先物キャピタルゲイン＋T-Bill金利 | BPS/CCプレミアム＋T-Bill金利 | CC/BPSプレミアム＋T-Bill金利 |

※ Emergency時の共通原則：PB転換・CC中止（高度に関わらず適用）。

**高度変更条件**

* 管制指示（FOMC声明・ドットチャート）に明確な変化があった場合  
* 実質金利のトレンドが反転し、確定した場合  
* 制約：前回の変更から6ヶ月未満の場合は一時的なノイズとみなし高度を維持する  
* 高度変更確定時は5-3-2（CCパージ）を即時起動

**飛行制限（🔵事前制限）**

* 雇用統計・CPI・FOMC発表：24時間前〜1時間後はBoost禁止  
* メジャーSQ：木曜引け〜金曜はBoost禁止  
* 大統領選・債務上限期限：1週間前〜はBoost禁止  
* 上記以外の重要指標発表：48時間以内はBoost禁止  
* Boost稼働中の場合：Cruiseへダウン

### **2-2. 操縦制御（運用モード）**

**［概念］** パイロットがシステムへの介入レベルを決定する。デフォルトはマニュアルであり、信頼が積み上がるにつれて自動化の範囲を拡大する。Emergencyプロトコルのみは半自動以上で自動執行される。

**［仕様］**

| モード | 内容 | 切替方法 |
| ----- | ----- | ----- |
| 🟡 マニュアル（デフォルト） | システムが遷移内容を算出・提示。パイロットがTelegramで承認後に執行 | — |
| 🟠 半自動 | Emergencyプロトコルのみ自動執行。それ以外はマニュアルと同様 | 人間が明示的に操作して起動・解除 |
| 🟢 全自動 | フライトコントローラーのルールに基づきシステムが全て自動執行 | 人間が明示的に操作して起動・解除 |

**執行原則**

* 通常時：Adaptive Algo Patient  
* Emergency時：Adaptive Algo Urgent  
* ギアダウン時：先物削減→ブースターBPS削減→メインエンジンPB転換の順で執行  
* Emergencyプロトコル発動時：メインエンジンPB転換注文を即時発動（買いP追加を先行し裸売り時間をゼロにする）  
* 再編前はCheck Marginで価格-10%/IV+10%をシミュレーションする

### **2-3. ND/EICAS（補助計器）**

**［概念］** フライトコントローラーのロジックには介入しないが、パイロットが計器の故障や気流の変質を察知するために使用する。月次の高度・ルート変更判断における基礎資料となる。

**［仕様］**

日次で算出されTelegramに報告。SQLite等に保存。

| カテゴリ | 指標名 | メタファー名 | 診断内容 |
| ----- | ----- | ----- | ----- |
| 気象診断 | Basis Stress（GC \- XAUUSD） | 気圧配置図 | 先物・現物の分断。過去1年平均の3σ超でTelegram通知。フライトコントローラーへの自動注入なし |
| 航法診断 | Corr20（NQ/GC） | 偏流計 | 2銘柄の相関。想定した分散ルートからのドリフトを監視 |
| 機体診断 | Realized Skew（IV\_Put \- IV\_ATM） | 振動センサー | 下落警戒プレミアムの歪み。ICL上昇前の微振動を検知 |
| 機体診断 | Portfolio Vega（Σ Pos. Vega） | 耐熱シールド計 | ボラ急騰に対する防御力。V因子悪化時の機体ダメージを予測 |
| エンジン効率 | Theta/Margin（Total Theta / MM） | 燃費計 | インカム効率。機体負荷1単位あたりの時間価値回収量 |
| 航法記録 | Current DD（(NLV\_max \- NLV) / NLV\_max） | 高度ロス計 | 最高高度からの降下率。▲30%で墜落警報 |
| 航法記録 | Sharpe Ratio（R\_avg / σ） | 飛行品質計 | 揺れの少なさと上昇効率のバランス |

### **2-4. アビオニクス（警告・通知）**

**［概念］** フライトコントローラーが算出したEffective Levelをパイロットへ伝達する通知系。ADVISORYは情報共有、CAUTIONは減速指示、WARNINGは緊急介入を意味する。

**［仕様］**

| アラート階層 | メタファー | トリガー条件 | アクション |
| ----- | ----- | ----- | ----- |
| ADVISORY | 地上通信 | 補助計器の逸脱 | ログ記録およびTelegram通知 |
| CAUTION | マスターコーション | Effective Level \= 1 | 🟡Cruiseモードへ遷移。ブースターの新規建玉を停止 |
| WARNING | マスターワーニング | Effective Level \= 2 | 🔴Emergencyモードへ遷移。Emergencyプロトコルを執行 |

## **3\. フライトコントローラー**

**［概念］** フライトコントローラーは「個別・同期・制限」の3層からなる独立制御構造を採用し、各エンジンの出力を機械的に決定する。個別銘柄の波乱を個別制御層（ICL）が抑え、銘柄間の歩調の乱れを同期制御層（SCL）が検知し、最終層である制限制御層（LCL）が資本の安全限界を絶対的に保証する。上位層（LCL）の判断は下位の制御指令を無条件でオーバーライドし、いかなる市場環境下でも機体の生存を最優先する。

### **3-1. Layer1：Raw Data**

**［概念］** 未加工の市場・内部データ。計算・加工を一切行わない純粋な事実の保持。この層のデータが直接Layer4に触れることは禁止。

**［仕様］**

| ----- | ----- | ----- | ----- | ----- | ----- |

| GC1\! | 連続足・先物 | 13:30 | 翌2:30 | 翌3:30 | シグナル判定用。発注・資産計算は直近限月 |
| VXN | ボラティリティ指数 | 16:00 | 翌5:00 | 翌6:00 | V因子（NQ系）用 |
| GVZ | ボラティリティ指数 | 16:00 | 翌5:00 | 翌6:00 | V因子（GC系）用 |
| HYG | ETF | 16:00 | 翌5:00 | 翌6:00 | C因子用 |
| LQD | ETF | 16:00 | 翌5:00 | 翌6:00 | C因子用 |

**［仕様］**

先物はSettlement Price、ボラティリティ指数・ETFはClose Priceを使用する。SMA・高値等の計算も同様。計算は営業日ベース。

### **3-3. Layer3：Factors**

**［概念］** Layer2のSignalを購読し、独自の閾値テーブルに従って0/1/2のレベルを出力する単機能の計器群。各因子は自分がどの制御レベルに使われるかを知らない（0-1-Ⅷ）。悪化は即発動、復帰は条件を要求するという非対称性がヒステリシス原則（0-1-Ⅳ）の実装である。

---

#### **P因子（Price Stress）**

**［概念］** 機体の姿勢を示す。SMA20からの乖離・下落速度・高値からの距離の3軸で価格ストレスを評価する。3軸すべてが正常な時のみP0とし、1軸でも異常があればP1以上に昇格する。

**［仕様］**

NQ・GC各銘柄に対して独立して算出。

**NQ**

| Level | NQ\_日次変動率 / NQ\_累積変動率 | NQ\_過去20日高値乖離率 | NQ\_Trend |
| ----- | ----- | ----- | ----- |
| P0（Calm） | 日次 ±1.5%以内 AND 5日累積 \-3%以内 | \> ▲3% | Uptrend |
| P1（Pressure） | 日次 \-1.5〜-3% OR 5日累積 \-3〜-6% | ▲3%〜▲5% | — |
| P2（Shock） | 日次 \-3%以上 OR 2日累積 \-5%以上 | \< ▲5% | Downtrend（乖離率起因時のみ） |

**GC**

| Level | GC\_日次変動率 / GC\_累積変動率 | GC\_過去20日高値乖離率 | GC\_Trend |
| ----- | ----- | ----- | ----- |
| P0（Calm） | 日次 ±1.5%以内 AND 5日累積 \-3%以内 | \> ▲2.5% | Uptrend |
| P1（Pressure） | 日次 \-1.5〜-3% OR 5日累積 \-3〜-6% | ▲2.5%〜▲4% | — |
| P2（Shock） | 日次 \-3%以上 OR 2日累積 \-5%以上 | \< ▲4% | Downtrend（乖離率起因時のみ） |

**判定ルール**

* P0：3列全てを満たす  
* P1：日次変動率条件 OR 過去20日高値乖離率条件のいずれか  
* P2：日次変動率条件のいずれか OR（過去20日高値乖離率条件 AND Trend条件）

非対称性：悪化は即発動。復帰は各条件の解消を要する。

---

#### **V因子（Volatility Stress）**

**［概念］** 対気速度計。VXN/GVZが示す気流の速度（ボラ）が過速度になると機体が振動し、減速を強いられる。復帰時の1hノックインは、ボラが下がったことを日中の値動きで実際に確認するための安全装置である。

**［仕様］**

NQ系はVXN、GC系はGVZを参照。高高度・中高度と低高度で閾値テーブルが切り替わる。

|  | 高高度・中高度 | 低高度 |
| ----- | ----- | ----- |
| V2発動（即） | 40以上 | 35以上 |
| V2→V1復帰（2日確認） | 38未満 | 33未満 |
| V1発動（即） | 30以上 | 25以上 |
| V1→V0復帰（1日確認＋1hノックイン） | 28未満 | 23未満 |

非対称性：悪化方向はClose確定で即発動。復帰は確認日数を要する。

**1hノックイン（全高度 V1→V0復帰時）**

Step 1: Closeが閾値未満を1日確認  
Step 2: 翌営業日のコアタイム開始から監視開始（2本目以降の足を対象）。  
        コアタイム終了までのいずれかの1h足確定時点で  
        NQ系：VXN\_1h足ノックイン判定（Close \< 前日Close AND 陰線）  
        GC系：GVZ\_1h足ノックイン判定（Close \< 前日Close AND 陰線）  
            → V0復帰確定・執行  
        終日条件未達  
            → 執行見送り（翌日再判定）

---

#### **C因子（Credit Stress：NQ系）**

**［概念］** 昇降計（NQ用）。HYG/LQDが示す信用市場の血流を監視する。信用収縮はNQ失速の先行指標であり、SMA20割れまたは急落で即座にC2に昇格する。C1を設けず二択にすることで、判定の曖昧さを排除する。

**［仕様］**

HYG・LQDそれぞれ独立にSMA20乖離率・日次変動率を算出。C1廃止。C0またはC2の二択。高高度・中高度・低高度で閾値共通。

|  | 条件 |
| ----- | ----- |
| C2発動（即） | HYGまたはLQDの少なくとも一方で、HYG/LQD\_SMA20乖離率 \< 0 OR HYG/LQD\_日次変動率 ≤ \-2.5% |
| C0復帰（2日確認） | HYGおよびLQDの両方で、HYG\_SMA20乖離率 ≥ 0 AND LQD\_SMA20乖離率 ≥ 0を維持 |

非対称性：C2は閾値接触で即発動。C0復帰は2日間の静止を要求。

---

#### **R因子（Real-Rate Stress：GC系）**

**［概念］** 昇降計（GC用）。TIPが示す実質金利水準を監視する。TIP Closeの下落（実質金利上昇）はGC保有コストの増大を意味し、GCの優位性が失われる前兆となる。

**［仕様］**

TIP\_過去20日高値乖離率を参照。R1廃止。R0またはR2の二択。

|  | 高高度・中高度 | 低高度 |
| ----- | ----- | ----- |
| R2発動（即） | TIP\_過去20日高値乖離率 \< ▲2.5% | TIP\_過去20日高値乖離率 \< ▲1.5% |
| R0復帰（2日確認） | TIP\_過去20日高値乖離率 ≥ ▲1.5%を維持 | TIP\_過去20日高値乖離率 ≥ ▲0.5%を維持 |

非対称性：R2は閾値接触で即発動。R0復帰は2日間の静止を要求。

---

#### **T因子（Trend Sync）**

**［概念］** 旋回計。NQとGCという左右エンジンのトレンドが同期して下落し始めると、分散の効果が崩壊し機体全体が危険にさらされる。片肺飛行（T=1）でも警戒を要するが、両エンジン同時下落（T=2）は即時対応を要する。

**［仕様］**

NQ\_Trend・GC\_TrendシグナルをSCLへの入力値として機能する。

| T値 | NQ\_Trend | GC\_Trend | 意味 |
| ----- | ----- | ----- | ----- |
| 0 | Uptrend or Flat | Uptrend or Flat | 両銘柄安定 |
| 1 | Downtrend XOR Flat/Up | — | 片銘柄下落。歩調の乱れ |
| 2 | Downtrend | Downtrend | 両銘柄下落。分散崩壊 |

非対称性：悪化は即発動。T0復帰は両銘柄ともUptrend or Flatの確認を要する。

---

#### **U因子（Utilization：MM/NLV）**

**［概念］** Gメーター。NLVというフレームに対してポジションがかける構造応力を測定する。50%超は機体破損の臨界点であり、即時パージを要する。リアルタイム計算によりステートレスに判定し、バッファ幅のみでヒステリシスを実現する。

**［仕様］**

|  | 発動（即） | 復帰（即） |
| ----- | ----- | ----- |
| 2 | MM/NLV比 ≥ 50% | MM/NLV比 \< 43% |
| 1 | MM/NLV比 ≥ 40% | MM/NLV比 \< 36% |

---

#### **S因子（SPAN乖離率）**

**［概念］** タコメーター。SPAN証拠金が基準密度から乖離するほど、エンジンが過負荷状態にあることを示す。月次で基準密度を校正することで、相場環境の変化による自然な密度変化と異常な乖離を区別する。リアルタイム計算によりステートレスに判定し、バッファ幅のみでヒステリシスを実現する。

**［仕様］**

S \= 証拠金密度 / 基準証拠金密度

基準密度 \= 月次メンテナンス時に校正・保存した定数（→ 5-1参照）

|  | 発動（即） | 復帰（即） |
| ----- | ----- | ----- |
| 2 | S ≥ 1.3（小数点第二位切上げ） | S \< 1.3（小数点第二位切捨て） |
| 1 | S ≥ 1.1（小数点第二位切上げ） | S \< 1.1（小数点第二位切捨て） |

---

### **3-4. Layer4：Control Levels**

**［概念］** Layer3の因子判定を統合し、機体全体の飛行許可レベルを決定する最終命令層。フライトコントローラーはEffective Level以外の下位情報を参照しない（0-1-Ⅷ）。市場がどれほど好調でも、LCLが資本の限界を超えたと判断した瞬間に全出力を遮断する。

**［仕様］**

Layer2 Signals（NQ1\!/GC1\! Settlement・VXN/GVZ/HYG/LQD/TIP Close基準）  
        ↓  
Layer3 Factors  
ICL\_NQ \= max(P\_NQ, V\_NQ, C)   ← NQ個別環境評価  
ICL\_GC \= max(P\_GC, V\_GC, R)   ← GC個別環境評価  
SCL    \= T因子相関ロジック      ← 銘柄間同期評価  
LCL    \= max(U, S)             ← 内部資本制約評価  
        ↓  
Effective Level \= max(ICL\_NQ, ICL\_GC, SCL, LCL)  
        ↓  
スロットルモード確定（Boost / Cruise / Emergency）

| Effective Level | スロットルモード | 気流状態 | 意味 |
| ----- | ----- | ----- | ----- |
| 0 | Boost | 安定気流 | 層流。機体が安定しており最大推力を許容 |
| 1 | Cruise | 不安定気流 | 擾乱。機体が振動し始め安全のため減速を要する |
| 2 | Emergency | 失速気流／限界負荷 | 揚力喪失または臨界点。即時パージを要する |

**ICL**

ICL\_NQ \= max(P\_NQ, V\_NQ, C)  
ICL\_GC \= max(P\_GC, V\_GC, R)

非対称性：悪化は即発動。改善は各因子の復帰条件に従う。

**SCL**

SCL \= T因子の値をそのまま採用

| SCL | 状態 | 意味 |
| ----- | ----- | ----- |
| 0 | 両安定 | 分散機能している |
| 1 | 片Downtrend | 片肺飛行。歩調の乱れを検知 |
| 2 | 両Downtrend | 分散崩壊。全系警戒 |

**LCL**

LCL \= max(U, S)

内部資本制約を評価する。市場環境に関わらず最終許可層として機能し、LCL=2は全出力を強制遮断する。非対称性：悪化は即発動。改善は各因子の復帰条件に従う。

**拡張ガイド：新指標追加時の手順**

将来新しい指標を導入する場合、変更はLayer2とLayer3のみに限定される。Layer4のコードは一切変更しない。これが階層分離原則（0-1-Ⅷ）の実装上の保証である。

## **4\. 操縦翼面（制御・防衛システム）**

航空機の操縦翼面は、空気抵抗（ボラティリティ・時間価値）を制御し、機体の姿勢と高度を維持するための動的デバイスである。

### **4-1. CC（スポイラー / スピードブレーキ）**

**［概念］** 機体が浮き上がりすぎる（急激な踏み上げ）のを抑え、気流をインカムとして回収する揚力抑制デバイス。順風（P0）や高高度では抵抗を減らすため格納するが、低高度（利上げ期）では積極的に展開して燃費（インカム効率）を高める。失速（Effective=2）時には機体の立て直しを阻害する不要な抵抗となるため、真っ先に格納（パージ）させる。

**［仕様］**

※ 高高度（利下げサイクル）ではCCを使用しない。中高度・低高度時のみ適用。 ※ メインエンジン層専用。姿勢制御エンジン・ブースターへの装着不可（1-3参照）。

**CC × スロットルモード マトリクス**

| ユニット | Cruise | Emergency |
| ----- | ----- | ----- |
| 姿勢制御エンジン | 装着不可 | 装着不可 |
| メインエンジン | 稼働 | 停止 |
| ブースター | — | — |

**基本パラメータ（Cruise・メインエンジン稼働時）**

| 項目 | 高高度 | 中高度 | 低高度 |
| ----- | ----- | ----- | ----- |
| 主目的 | — | バランス型インカム＋踏み上げ警戒 | 証拠金温存＋確実なインカム |
| 満期（DTE） | — | Monthly（30〜45日前後） | Monthly（30〜45日前後） |
| 枚数比率 | — | 100%固定（対先物） | 100%固定（対先物） |
| ターゲットΔ | — | 0.15 | 0.25 |
| P0（強気）補正 | — | CC停止 | Delta 0.15（踏み上げ防止） |

**状態遷移マトリクス**

| 状態（Effective × 高度） | 中高度 | 低高度 |
| ----- | ----- | ----- |
| Effective=0（P0含む） | Delta 0.15 / Monthly（P0時停止） | Delta 0.25 / Monthly（P0時Delta 0.15） |
| Effective=1 | Delta 0.15 / Monthly | Delta 0.20 / Monthly |
| LCL=1 | メインエンジン停止 | メインエンジン停止 |
| Effective=2 | 全CC停止 | 全CC停止 |

### **4-2. BPS（フラップ / エレボン）**

### **4-3. PB（防氷・耐熱外装＆逆噴射装置）**

**［概念］** 通常巡航時は氷結や落雷（暴落予兆）から機体を守る防氷・耐熱外装として静的防衛を担う。普段は燃費を削るだけのコストだが、安全圏を確保するための生命線。緊急時（Effective=2）は逆噴射装置（スラストリバーサー）として機能し、エンジンの排気を斜め前方に跳ね返して先物の強力な慣性（下落エネルギー）を物理的に押し戻し墜落を阻止する。メインエンジン層および姿勢制御エンジン層に装備される。ブースター層への装着不可（1-3参照）。

### **5-2. Emergencyプロトコル**

**［概念］** スロットルがEmergencyに遷移した瞬間、全ユニットで同時に防衛態勢へ移行する。処理の順序ではなく同時性が重要であり、裸売りの時間をゼロにすることが最優先である。

**［仕様］**

スロットルがEmergencyに遷移した場合、以下を全ユニット同時に実行する。🔴強制自動・事後報告。

| ユニット | 対象 | アクション | 効果 |
| ----- | ----- | ----- | ----- |
| メインエンジン | BPS | K2を1枚追加しPB転換（1:2構成） | デルタ反転。下落をNLV上昇に変換。裸売り時間ゼロ |
| メインエンジン | K1（売りP） | 即時買戻し | 最大デルタ要因の売りを消去。Long Put 2枚状態へ |
| メインエンジン | CC（売りC） | 即時買戻し | Shortポジションが消滅 |
| ブースター | 先物 | 即時決済 | 証拠金要求の最大要因を最速で解放 |
| 姿勢制御エンジン | K1（売りP） | 即時買戻し | 売りPを消滅。無限損失リスクをゼロ化 |
| 姿勢制御エンジン | 先物 | 全決済 | 証拠金をさらに解放。K2を純粋な利益源泉へ転換 |

### **5-3. 随時タスク**

#### **5-3-1. PB利益確定**

**［概念］** 暴落後にボラが低下し、PBのデルタが縮小した局面で利益を確定する。Effective=2継続中は決済禁止とし、防衛機能を維持することを優先する。

**［仕様］**

監視指標：NQ系→VXN Close。GC系→GVZ Close。IVピーク値＝直近15営業日の最大値（当日除く）。

パターンA（デルタレンジ縮小）：全ポジションを🟡決済承認フローへ

* 現在のデルタ ≤ 初期構築時の50%  
* 含み益 ≥ \+20%

パターンB（ボラ正常化）：50〜100%を🟡決済承認フローへ

* 現在のVXN/GVZ Close ≤ IVピーク値×0.85  
* 含み益 ≥ \+40%  
* 残存DTE ≥ 7日

執行禁止条件：Effective=2継続中 OR 含み益 \< \+20%

#### **5-3-2. CCパージ**

**［概念］** 高度変更またはEmergency発動時に、CCを全て撤収する。高度変更時はインカムの性格が変わるため、旧高度のCCをそのまま残すことは構造汚染となる。

**［仕様］**

以下のいずれかのトリガー発動時に実行する。

**トリガー**

* 高度変更確定時（CCを廃止する方向への移行）  
* Effective=2発動時  
* Emergencyプロトコル発動時

**処理シーケンス**

1. 未約定CC注文を全キャンセル  
2. 既存ポジションをDelta降順（高い順）に決済  
   * Delta ≥ 0.10 → Adaptive Algo Urgent  
   * Delta \< 0.10 → Adaptive Algo Patient  
   * ITM → 即時クローズ（先物含み益との相殺を考慮）  
3. ロールアップ禁止：パージ実行後はトリガー解除まで CC再エントリー禁止

#### **5-3-3. 現金バッファ管理**

**［仕様］**

* 目標残高：$150k  
* $30k割れ時：T-Bill一部売却提案→承認後執行  
* 現金$100k以上を日次確認
