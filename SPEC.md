# **マクロインカム戦略 完全運用定義書**

---

## **0\. 戦略概要**

### **0-1. システム憲章**

**Ⅰ. 目的** 本システムは、資本の長期的存続と複利成長を目的とする。 最大化は目指すが、破壊を伴う最大化は拒否する。

**Ⅱ. 設計原則** 本システムは状態遷移制御系である。 エンジン出力は3段階とする。 連続係数は最終的に離散状態へ収束する。 抽象レイヤーは混合しない。

**Ⅲ. 抽象構造** 市場はICL・SCLに抽象化される。 資本はLCLに抽象化される。 市場因子は直接資本を制御しない。 ICL / SCL / LCL → Effective Level → Engine この順序を破らない。

**Ⅳ. ヒステリシス原則** 状態は瞬間値で決定しない。 遷移には持続条件を必要とする。 復帰は進入より遅くてよい。

**Ⅴ. 美学原則** 不要な自由度を増やさない。 例外を作らない。 局所最適のために構造を壊さない。 強度はレイヤー内で解決する。

**Ⅵ. 実用原則** 暴落時に自己改造しない。 勝ち相場で構造を緩めない。 市場環境の変化はパラメータで吸収する。 レイヤーは固定資産である。

**Ⅶ. 禁則事項** 市場因子を直接LCLへ注入しない。 恐怖を理由に例外処理を追加しない。 係数の微調整で本質を誤魔化さない。

**Ⅷ. 階層分離原則** 各層は自分より上位の層の意図を知ってはならない。 因子は自身がどの制御レベルに使われるかを知らない。 OSCoreはEffective Level以外の下位情報を参照しない。 層をまたいだ直接参照は構造汚染である。

### **0-3. 運用の基本原則**

本戦略は、フライトプランを指揮系統の起点とし、計器が取得した情報をOSが統合して運用判断を機械的に決定する。

**運用判断の構造**

| 名称 | 役割 | 実装・ロジック |
| ----- | ----- | ----- |
| フライトプラン | 戦略的介入と環境認識 | 金利サイクルや重要イベントに基づき、パラメータの切り替えやリスク許容量（スロットル）の抑制を決定する |
| 計器 | 外部・内部情報の取得 | 市場環境（外部）と証拠金状況（内部）を観測。非対称な判定閾値により、昇格時のみバッファや継続期間を要求し、判断のバタつきを防止する |
| OS | 情報の統合と機械的執行 | 計器が取得した情報を評価し、最悪のシグナルを即採用。生存を最優先し、運用の可否・規模を非情に決定する |

**変数の定義と統合プロセス（OS層）**

OSは、計器から送られる「市場環境」と「自己の体力」の情報を3層で評価し、最も保守的な判断を優先する。

■ 個別制御（ICL）：個別銘柄の環境評価 $$ICL\_{NQ} \= \\max(P, V, C)$$ $$ICL\_{GC} \= \\max(P, V, R)$$ （$P$: 価格, $V$: ボラティリティ, $C$: 信用, $R$: 実質金利）

■ 同期制御（SCL）：銘柄間トレンド相関評価 $$SCL \= T\\text{（相関ロジック）}$$ （$T$: NQ/GC銘柄間トレンド同期）

■ 制限制御（LCL）：資本の安全限界評価 $$LCL \= \\max(U, S)$$ （$U$: 証拠金使用率, $S$: SPAN乖離率）

■ 最終命令 $$Effective \= \\max(ICL\_{NQ}, ICL\_{GC}, SCL, LCL)$$

**非対称性の徹底**

* 降格（悪化時）：異常を検知した瞬間、計器は即座にOSへ通知し、OSはノータイムで運用を縮退・停止させる  
* 昇格（改善時）：計器層で設定された「継続期間」や「一定以上の回復幅」を満たさない限り、上位判定への復帰を認めない

**因子体系一覧**

| 区分 | 制御レベル | 計器名 | 観測項目 | 因子名 |
| ----- | ----- | ----- | ----- | ----- |
| 外部/個別（NQ） | ICL | 姿勢指示器 | 価格・移動平均 | Price（P） |
| 外部/個別（NQ） | ICL | 対気速度計 | ボラティリティ（VXN） | Volatility（V） |
| 外部/個別（NQ） | ICL | 昇降計（NQ用） | 信用スプレッド（HYG/LQD） | Credit（C） |
| 外部/個別（GC） | ICL | 姿勢指示器 | 価格・移動平均 | Price（P） |
| 外部/個別（GC） | ICL | 対気速度計 | ボラティリティ（GVZ） | Volatility（V） |
| 外部/個別（GC） | ICL | 昇降計（GC用） | 実質金利（TIP） | Real-Rate（R） |
| 外部/同期 | SCL | 旋回計 | 2銘柄同期トレンド | Trend（T） |
| 内部/制限 | LCL | Gメーター | 証拠金使用率（MM/NLV） | Utilization（U） |
| 内部/制限 | LCL | タコメーター | SPAN乖離率 | SPAN（S） |

## **1\. 航空機推進・制御システム（Aeronautics & Propulsion）**

本システムは、機体本体と、それに取り付けられた3つのエンジンポッドで構成される。各ポッドは、推進力（先物）と制御面（オプション）が一体化した独立駆動ユニットである。

### **1-1. ユニット構成**

| ユニット名 | 推進（先物） | 制御（OP） | 役割の論理 |
| ----- | ----- | ----- | ----- |
| メインエンジン | Mini先物（50%） | 防衛装甲（PB/BPS/CC） | 機体屋台骨：墜落を阻止し生還を担う中枢。PB・BPS・CCをすべて装備 |
| 姿勢制御エンジン | Micro先物（25%） | 逆噴射（PB） | 主力エンジン：巡航の要。異常時は逆噴射（PB）で機体を急減速させる |
| ブースター | Micro先物（25%） | 加速フラップ（BPS） | 補助エンジン：加速専用。リスク発生時は即座に停止（エンジンカット）する |

### **1-2. 燃料と油圧（資産基盤）**

すべての運用のエネルギー源となる純資産（NLV $1M）。

| 項目 | 形態 | 金額 | 役割 |
| ----- | ----- | ----- | ----- |
| 燃料タンク | T-Bill（1年物） | $850k | 航続距離：担保価値（約94%）維持。3ヶ月ラダー運用 |
| 油圧系統（現金） | 現金バッファ | $150k | 即時動作：値洗い（油圧）原資。$30k割れで燃料タンク（T-Bill）から補給 |

### **1-3. ユニット動力機構（設計図・Blueprint）**

各エンジンはNQ専用またはGC専用の1インスタンスであり、層ごとの設計図（LayerBlueprint）に基づいて3層（メイン・姿勢制御・ブースター）で構成される。設計図はTOML等で定義し、起動時に読み込んで実行中は変更しない（0-1-Ⅵ）。

* **メインエンジン層**：Mini先物（NQ/GC）＋ PB ＋ CC ＋ BPS。設計図はMain用Blueprint

* **姿勢制御エンジン層**：Micro先物（MNQ/MGC）＋ PB。CC・BPSは装着不可（5-1, 5-2）。設計図はAttitude用Blueprint

* **ブースター層**：Micro先物（MNQ/MGC）＋ BPS。PB・CCは装着不可。設計図はBooster用Blueprint

* **構成**：層ごとに「先物 ＋ 当該層で装着する操縦翼面」をBlueprintの比率で保持する。メイン層のみがPB/CC/BPSをすべて持つ

* **対称配分（1:1）**：NQ用エンジンとGC用エンジンを同構造で組み立て、NQ系列とGC系列のエクスポージャを常に等価に保つ

* **最大出力上限**：NLV $1Mに対し、総エクスポージャ $2M（200%）を上限とする

### **1-4. スロットルモードとパージ・シーケンス**

OSが決定するモードにより、物理的なエンジンカット順序を制御する。

**Boost（200%）：全発エンジン稼働**

* 構成：メインエンジン＋姿勢制御エンジン＋ブースター  
* 状態：追い風を利用した最大加速。全リソースを推進に投下

**Cruise（150%）：ブースター・エンジンカット**

* 構成：メインエンジン＋姿勢制御エンジン  
* 状態：気流の乱れ（Effective=1）でブースター（Micro先物）をカット。逆噴射（PB）を備えた安定航行へ

**Emergency（100%）：姿勢制御エンジン・パージ＆逆噴射全開**

* 構成：メインエンジンのみ  
* 状態：重大な危機（Effective=2）。本体以外の先物を全パージ。防衛装甲（PB）で生存を図る

### **1-5. OSによる執行：出力の遮断と復帰**

OSは計器の状態に基づき、以下のプロセスを機械的に執行する。

**出力遮断（パージ）：** 計器が「レッドライン」に達した際、OSは即座に外側から切り離す。

1. ブースターをエンジンカット（200% → 150%）  
2. 姿勢制御エンジンをパージ（150% → 100%）

**再点火（環境復帰）：** 「昇格条件」を満たした場合にのみ、メインエンジンに近いユニットから順に再点火を許可する。

1. 姿勢制御エンジンを再点火（100% → 150%：Cruise復帰）  
2. ブースターを再点火（150% → 200%：Boost復帰）

## **2\. フライトプラン**

本戦略では金利サイクルを「巡航高度」、運用方針を「ルート」として定義する。月次でドットチャート・FF金利見通し・重要イベントを確認し、高度・ルートを設定する。高度を変更した場合は原則6ヶ月間維持する。

高度の変更は操縦翼面（5-1〜5-3）のパラメータのみに作用する。高高度・中高度の差異は操縦翼面のパラメータに限定する。計器（3）およびOS判定（4）は高度によらず共通。ただしV因子・C因子・R因子の閾値テーブル切り替えを例外とし、高高度・中高度と低高度で異なる定規を適用する。

| 項目 | 高高度（利下げサイクル） | 中高度（安定期） | 低高度（利上げサイクル） |
| ----- | ----- | ----- | ----- |
| スロットル上限 | Boost（200%） | Boost（200%） | Cruise（150%） |
| 基本方針 | 上昇追従。先物成長＋BPSインカム | バランス型。時間価値回収＋踏み上げ警戒 | 防御優先。証拠金温存＋確実なインカム |
| CC方針 | なし（高高度では廃止） | メインエンジン稼働。P0時停止 | セータ重視。確実なプレミアム回収 |
| BPS方針 | OTM浅め（▲5〜7%） | OTM中間（▲7〜8%） | OTM深め（▲8〜10%） |
| PB方針 | 感度高め（Δ0.12） | 中間（Δ0.10） | コスト抑制（Δ0.08） |
| 主な収益源 | 先物キャピタルゲイン＋T-Bill金利 | BPS/CCプレミアム＋T-Bill金利 | CC/BPSプレミアム＋T-Bill金利 |

※ Emergency時の共通原則：PB転換・CC中止（高度に関わらず適用）。

**高度設定時の確認項目**

* 管制指示（FOMC声明・ドットチャート）に明確な変化があった場合  
* 実質金利のトレンドが反転し、確定した場合  
* 制約：前回の変更から6ヶ月未満の場合は一時的なノイズとみなし高度を維持する  
* 高度変更確定時は6-3-2（CCパージ）を即時起動

**飛行制限（🔵事前制限）**

* 雇用統計・CPI・FOMC発表：24時間前〜1時間後はBoost禁止  
* メジャーSQ：木曜引け〜金曜はBoost禁止  
* 大統領選・債務上限期限：1週間前〜はBoost禁止  
* 上記以外の重要指標発表：48時間以内はBoost禁止  
* Boost稼働中の場合：Cruiseへダウン

## **3\. コックピット（監視・判断）**

### **3-1. PFD（Primary Flight Display）：主計器**

OSのアルゴリズムに直接接続され、スロットルモードを決定する。シグナル判定は連続足（NQ1\!/GC1\!）を使用。発注・資産計算は直近限月（Front Month）を使用。終値はNY現物クローズ（ET 16:00）を基準とする。

| 系統 | 制御レベル | メタファー名 | 指標（OS変数） | 役割と挙動 |
| ----- | ----- | ----- | ----- | ----- |
| 外部/個別（NQ） | ICL | 姿勢指示器 | 価格 / SMA20（NQ1\!） | P因子：SMA20に対する機体の傾き。安定した上昇・水平維持を監視 |
| 外部/個別（NQ） | ICL | 対気速度計 | VXN | V因子：気流の速度（ボラ）。過速度は擾乱を引き起こし減速を強いる |
| 外部/個別（NQ） | ICL | 昇降計（NQ用） | HYG / LQD | C因子：信用収縮の検知。血流（信用）の停滞はNQ失速の前兆 |
| 外部/個別（GC） | ICL | 姿勢指示器 | 価格 / SMA20（GC1\!） | P因子：SMA20に対する機体の傾き。安定した上昇・水平維持を監視 |
| 外部/個別（GC） | ICL | 対気速度計 | GVZ | V因子：気流の速度（ボラ）。過速度は擾乱を引き起こし減速を強いる |
| 外部/個別（GC） | ICL | 昇降計（GC用） | TIP | R因子：実質金利の検知。金利上昇はGC保有コスト増大の予兆 |
| 外部/同期 | SCL | 旋回計 | NQ1\!/GC1\! 2銘柄トレンド | T因子：左右エンジンの推力バランス。同期的な下降気流は全系停止の予兆 |
| 内部/制限 | LCL | Gメーター | MM / NLV | U因子：構造応力。NLVというフレームに対しポジションがかける圧力 |
| 内部/制限 | LCL | タコメーター | SPAN乖離率 | S因子：駆動負荷。SPAN乖離率を監視しオーバーヒートを防ぐ |

### **3-2. ND/EICAS（補助計器）：システム診断・予報**

OSのロジックには介入しないが、パイロットが計器の故障や気流の変質を察知するために使用する。日次で算出されTelegramに報告。SQLite等に保存し月次の高度・ルート変更確認における基礎資料とする。

| カテゴリ | 指標名 | メタファー名 | 診断内容 |
| ----- | ----- | ----- | ----- |
| 気象診断 | Basis Stress（GC \- XAUUSD） | 気圧配置図 | 先物・現物の分断。過去1年平均の3σ超でTelegram通知。OSへの自動注入なし |
| 航法診断 | Corr20（NQ/GC） | 偏流計 | 2銘柄の相関。想定した分散ルートからのドリフトを監視 |
| 機体診断 | Realized Skew（IV\_Put \- IV\_ATM） | 振動センサー | 下落警戒プレミアムの歪み。ICL上昇前の微振動を検知 |
| 機体診断 | Portfolio Vega（Σ Pos. Vega） | 耐熱シールド計 | ボラ急騰に対する防御力。V因子悪化時の機体ダメージを予測 |
| エンジン効率 | Theta/Margin（Total Theta / MM） | 燃費計 | インカム効率。機体負荷1単位あたりの時間価値回収量 |
| 航法記録 | Current DD（(NLV\_max \- NLV) / NLV\_max） | 高度ロス計 | 最高高度からの降下率。▲30%で墜落警報 |
| 航法記録 | Sharpe Ratio（R\_avg / σ） | 飛行品質計 | 揺れの少なさと上昇効率のバランス |

| 🟡 マニュアル（デフォルト） | システムが遷移内容を算出・提示。パイロットがTelegramで承認後に執行 | — |
| 🟠 半自動 | Emergencyプロトコルのみ自動執行。それ以外はマニュアルと同様 | 人間が明示的に操作して起動・解除 |
| 🟢 全自動 | OSルールに基づきシステムが全て自動執行 | 人間が明示的に操作して起動・解除 |

**執行原則**

* 通常時：Adaptive Algo Patient  
* Emergency時：Adaptive Algo Urgent  
* ギアダウン時：先物削減→ブースターBPS削減→メインエンジンPB転換の順で執行  
* Emergencyプロトコル発動時：メインエンジンPB転換注文を即時発動（買いP追加を先行し裸売り時間をゼロにする）  
* 再編前はCheck Marginで価格-10%/IV+10%をシミュレーションする

### **4-2. OS構造**

本戦略のOSは、「個別・同期・制限」の3層からなる独立制御構造を採用し、各エンジンの出力を機械的に決定する。個別銘柄の波乱を個別制御層（ICL）が抑え、銘柄間の歩調の乱れを同期制御層（SCL）が検知し、最終層である制限制御層（LCL）が資本の安全限界を絶対的に保証する。上位層（LCL）の判断は下位の制御指令を無条件でオーバーライドし、いかなる市場環境下でも機体の生存を最優先する。

**情報の階層構造（Data Hierarchy）**

| 階層 | 名称 | 内容 | 役割 |
| ----- | ----- | ----- | ----- |
| Layer 1 | Raw Data | 終値・出来高・IV・金利・証拠金・VIX | 未加工の市場・内部データ |
| Layer 2 | Signals | トレンド（Up/Down/Flat）・日次変動率・累積変動率・Hourlyショック | 統計的加工を施した共通部品。複数の因子が共有参照する |
| Layer 3 | Factors | P / V / C / R / T / U / S因子（各0/1/2判定） | 特定の観点での状態判定。制御レベルへの入力値 |
| Layer 4 | Control Levels | ICL / SCL / LCL → Effective Level | エンジンを操作するための最終命令レベル |

**各層の実装ルールと制限**

**Layer 1：Raw Data（生データ層）**

* 実装内容：API（IBKR等）から取得した直後のOHLCV・IV・金利・証拠金額  
* ルール：計算・加工を一切行わない純粋な事実の保持  
* 制限：この層のデータが直接Layer 4（OSCore）に触れることは禁止

**Layer 2：Signals（共通シグナル層）**

* 実装内容：トレンド（Up/Down/Flat）・日次変動率・Downside Gap・累積変動率・1h足陽線判定等  
* ルール：複数の因子が参照する共通言語への変換。トレンド判定を一箇所で行うことでP因子とSCLの間でトレンド認識の不一致が物理的に起こらない。SMA20等の重い統計処理を一度で済ませ全因子に配る  
* 制限：レベル判定（0/1/2）はここでは行わず、あくまで数値や状態として保持する

**Layer 3：Factors（因子判定層）**

* 実装内容：P / V / C / R / T / U / S の各因子  
* ルール：Layer 2からシグナルを購読し、独自の閾値テーブルに従って0/1/2のレベルを出力する  
* 制限：因子は自分がどの銘柄のICLに使われるかやSCLとの兼ね合いを知ってはならない（0-1-Ⅷ）。各因子は割り当てられたシグナルのみを見て淡々とレベルを算出する単機能の計器に徹する

**Layer 4：Control Levels（制御レベル層）**

* 実装内容：ICL（Individual）・SCL（Synchronous）・LCL（Limit）  
* ルール：複数の因子の結果を統合しEffective Levelを算出してエンジンの出力を決定する。`ICL_NQ=max(P,V,C)`・`ICL_GC=max(P,V,R)`・`SCL=T相関ロジック`・`LCL=max(U,S)`  
* 制限：ここで確定したEffective Level以外、OSCoreは下位レイヤーの情報を参照してはならない（0-1-Ⅷ）

**拡張ガイド：新指標追加時の手順**

将来新しい指標を導入する場合、変更はLayer 2とLayer 3のみに限定される。Layer 4（ICL/SCL/LCL）およびOSCoreのコードは一切変更しない。例：新ボラティリティ指標の追加→Layer 2に新Signalを追加→Layer 3のV因子判定ロジックを更新。これが階層分離原則（0-1-Ⅷ）の実装上の保証である。

**3層制御構造**

銘柄別シグナル（Layer2・連続足NQ1\!/GC1\!）を各因子（Layer3）へ供給  
        ↓  
ICL\_NQ \= max(P, V, C)   ← NQ個別環境評価  
ICL\_GC \= max(P, V, R)   ← GC個別環境評価  
SCL    \= T相関ロジック   ← 銘柄間同期評価  
LCL    \= max(U, S)      ← 内部資本制約評価  
        ↓  
Effective Level \= max(ICL\_NQ, ICL\_GC, SCL, LCL)  
        ↓  
スロットルモード確定（Boost / Cruise / Emergency）

**Effective Level × スロットルモード対応表**

| Effective Level | スロットルモード |
| ----- | ----- |
| 0 | Boost |
| 1 | Cruise |
| 2 | Emergency |

**制御レベルの意味**

| レベル | メタファー | 状態の意味 |
| ----- | ----- | ----- |
| 0 | 安定気流 | 層流。機体が安定しており最大推力を許容 |
| 1 | 不安定気流 | 擾乱。機体が振動し始め安全のため減速を要する |
| 2 | 失速気流／限界負荷 | 揚力喪失または臨界点。即時パージを要する |

### **4-2-1. ICL（Individual Control Level：個別制御層）**

個別銘柄の環境を評価する。NQ・GCそれぞれに対して独立して算出する。

ICL\_NQ \= max(P, V, C)  
ICL\_GC \= max(P, V, R)

非対称性：悪化は即発動。改善は各因子の復帰条件に従う。

#### **4-2-1-1. P因子（Price Stress）**

**共通定義**

シグナル判定：NQ1\!/GC1\!（連続足・Back-adjusted）  
終値定義：NY現物クローズ（ET 16:00）  
発注・資産計算：直近限月（Front Month）  
Downside Gap \= (終値 / 過去20日高値終値) \- 1（負値）  
トレンド定義 → Layer2 Signal参照（4-2-2）

**NQ**

| Level | 日次変動条件 | Downside Gap | トレンド |
| ----- | ----- | ----- | ----- |
| P0（Calm） | 日足変動 ±1.5%以内 AND 5日累積 \-3%以内 | \> ▲3% | Uptrend |
| P1（Pressure） | 日足 \-1.5〜-3% OR 5日累積 \-3〜-6% | ▲3%〜▲5% | — |
| P2（Shock） | 日足 \-3%以上 OR 2日累積 \-5%以上 | \< ▲5% | Downtrend（Gap起因時のみ） |

**GC**

| Level | 日次変動条件 | Downside Gap | トレンド |
| ----- | ----- | ----- | ----- |
| P0（Calm） | 日足変動 ±1.5%以内 AND 5日累積 \-3%以内 | \> ▲2.5% | Uptrend |
| P1（Pressure） | 日足 \-1.5〜-3% OR 5日累積 \-3〜-6% | ▲2.5%〜▲4% | — |
| P2（Shock） | 日足 \-3%以上 OR 2日累積 \-5%以上 | \< ▲4% | Downtrend（Gap起因時のみ） |

**判定ルール**

* P0：3列全てを満たす  
* P1：日次変動条件 OR Downside Gap条件のいずれか  
* P2：日次変動条件のいずれか OR（Downside Gap条件 AND トレンド条件）

#### **4-2-1-2. V因子（Volatility Stress）**

高高度・中高度と低高度で閾値テーブルが切り替わる。

|  | 高高度・中高度 | 低高度 |
| ----- | ----- | ----- |
| V2発動（即） | 40以上 | 35以上 |
| V2→V1復帰（2日確認） | 38未満 | 33未満 |
| V1発動（即） | 30以上 | 25以上 |
| V1→V0復帰 | 28未満（1日確認＋1hノックイン） | 23未満（1日確認＋1hノックイン） |

非対称性：悪化方向は終値（ET 16:00）確定で即発動。復帰は確認日数を要する。V1→V0復帰は全高度で1hノックインを適用。

**1hノックイン（全高度 V1→V0復帰時）**

Step 1: 終値（ET 16:00）が閾値未満を1日確認  
Step 2: 翌営業日のNY現物市場時間（ET 09:30〜16:00）内の  
        いずれかの1h足確定時点で  
        1h足終値 \> 前日ET16:00終値 AND 1h足が陽線  
            → V0復帰確定・執行  
        終日条件未達  
            → 執行見送り（翌日再判定）

#### **4-2-1-3. C因子（Credit Stress：NQ系）**

NQ系専用。高高度・中高度と低高度で閾値テーブルが切り替わる。C1廃止。C0またはC2の二択。

**HYG/LQD**

|  | 高高度・中高度 | 低高度 |
| ----- | ----- | ----- |
| C2発動（即） | SMA20を下回る OR 前日比-2.5%以上 | SMA20を下回る OR 前日比-2.5%以上 |
| C0復帰（2日確認） | SMA20以上を維持 | SMA20以上を維持 |

非対称性：C2は閾値接触で即発動。C0復帰は2日間の静止を要求。

#### **4-2-1-4. R因子（Real-Rate Stress：GC系）**

GC系専用。高高度・中高度と低高度で閾値テーブルが切り替わる。R1廃止。R0またはR2の二択。

**TIP**

|  | 高高度・中高度 | 低高度 |
| ----- | ----- | ----- |
| R2発動（即） | 高値比▲2.5%超 | 高値比▲1.5%超 |
| R0復帰（2日確認） | 高値比▲1.5%以内を維持 | 高値比▲0.5%以内を維持 |

非対称性：R2は閾値接触で即発動。R0復帰は2日間の静止を要求。

### **4-2-2. SCL（Synchronous Control Level：同期制御層）**

銘柄間のトレンド相関を評価する。NQ・GC双方のLayer2トレンドシグナルを突き合わせて算出する。

**トレンド定義（Layer2 Signal）**

シグナル判定：NQ1\!/GC1\!（連続足・Back-adjusted）  
終値定義：NY現物クローズ（ET 16:00）

Uptrend   \= 終値 \> SMA20 × 1.005  
Downtrend \= 終値 \< SMA20 × 0.995  
※ SMA20：過去20営業日終値の単純移動平均  
※ P因子・SCLが共通参照する共用シグナル（Layer2で一元計算）

**SCL判定ロジック**

if NQ=Downtrend and GC=Downtrend:  
    SCL \= 2  
elif NQ=Downtrend XOR GC=Downtrend:  
    SCL \= 1  
else:  
    SCL \= 0

| SCL | 状態 | 意味 |
| ----- | ----- | ----- |
| 0 | 両Uptrend | 分散機能している |
| 1 | 片Downtrend | 片肺飛行。歩調の乱れを検知 |
| 2 | 両Downtrend | 分散崩壊。全系警戒 |

非対称性：悪化は即発動。SCL0復帰は両銘柄ともUptrendの確認を要する。

---

### **4-2-3. LCL（Limit Control Level：制限制御層）**

内部資本制約を評価する。市場環境に関わらず最終許可層として機能し、LCL=2は全出力を強制遮断する。

LCL \= max(U, S)

非対称性：悪化は即発動。改善は各因子の復帰条件に従う。

#### **4-2-3-1. U因子（Gメーター：MM/NLV）**

|  | 発動（即） | 復帰（確認） |
| ----- | ----- | ----- |
| 2 | ≥ 50% | \< 45%（2日確認） |
| 1 | ≥ 40% | \< 38%（3日確認） |

#### **4-2-3-2. S因子（タコメーター：SPAN乖離率）**

S \= 現在の証拠金密度（Current Density）/ 基準証拠金密度（Base Density）

証拠金密度 \= MM ÷ (現在値 × 先物倍率)  
基準密度 \= 月次メンテナンス時に校正・保存した定数（→ 6-1参照）

|  | 発動（即） | 復帰（確認） |
| ----- | ----- | ----- |
| 2 | ≥ 1.3 | \< 1.2（2日確認） |
| 1 | ≥ 1.1 | \< 1.05（3日確認） |

## **5\. 操縦翼面（制御・防衛システム）**

航空機の操縦翼面は、空気抵抗（ボラティリティ・時間価値）を制御し、機体の姿勢と高度を維持するための動的デバイスである。

### **5-1. CC（スポイラー / スピードブレーキ）**

機体が浮き上がりすぎる（急激な踏み上げ）のを抑え、気流をインカムとして回収する揚力抑制デバイス。順風（P0）や高高度では抵抗を減らすため格納するが、低高度（利上げ期）では積極的に展開して燃費（インカム効率）を高める。失速（Effective=2）時には機体の立て直しを阻害する不要な抵抗となるため、真っ先に格納（パージ）させる。

※ 高高度（利下げサイクル）ではCCを使用しない。中高度・低高度時のみ適用。 ※ メインエンジン層専用。姿勢制御エンジン・ブースターへの装着不可（1-3参照）。

**CC × スロットルモード マトリクス**

| ユニット | Cruise | Emergency |
| ----- | ----- | ----- |
| 姿勢制御エンジン | 装着不可 | 装着不可 |
| メインエンジン | 稼働 | 停止 |
| ブースター | — | — |

**基本パラメータ（Cruise・メインエンジン稼働時）**

| 項目 | 高高度 | 中高度 | 低高度 |
| ----- | ----- | ----- | ----- |
| 主目的 | — | バランス型インカム＋踏み上げ警戒 | 証拠金温存＋確実なインカム |
| 満期（DTE） | — | Monthly（30〜45日前後） | Monthly（30〜45日前後） |
| 枚数比率 | — | 100%固定（対先物） | 100%固定（対先物） |
| ターゲットΔ | — | 0.15 | 0.25 |
| P0（強気）補正 | — | CC停止 | Delta 0.15（踏み上げ防止） |

**状態遷移マトリクス**

| 状態（Effective × 高度） | 中高度 | 低高度 |
| ----- | ----- | ----- |
| Effective=0（P0含む） | Delta 0.15 / Monthly（P0時停止） | Delta 0.25 / Monthly（P0時Delta 0.15） |
| Effective=1 | Delta 0.15 / Monthly | Delta 0.20 / Monthly |
| LCL=1（C1相当） | メインエンジン停止 | メインエンジン停止 |
| Effective=2 | 全CC停止 | 全CC停止 |

### **5-2. BPS（フラップ / エレボン）**

主翼の形状を変化させ、停滞相場や着陸態勢（低ボラ）でも十分な浮力を得るための標準翼面。空気の粘り（プレミアム）を味方につけて航続距離を伸ばす。メインエンジン層およびブースター層に装備される。姿勢制御エンジン層への装着不可（1-3参照）。

**BPS × スロットルモード マトリクス**

| ----- | ----- | ----- | ----- |
| 姿勢制御エンジン | 装着不可 | 装着不可 | 装着不可 |
| メインエンジン | 稼働 | 稼働 | （PB転換） |
| ブースター | 稼働 | — | — |

**高度別パラメータ**

| 項目 | 高高度 | 中高度 | 低高度 |

| 対象ユニット（Effective=0・SCL=0） | ブースター | ブースター | — |
| 枚数比率 | 1:1（対先物） | 1:1（対先物） | 1:1（対先物） |

* スプレッド幅：NQ/MNQ 200pt / GC/MGC $40

* BPSは損失限定のため約定優先度は低い

### **5-3. PB（防氷・耐熱外装＆逆噴射装置）**

通常巡航時は氷結や落雷（暴落予兆）から機体を守る防氷・耐熱外装として静的防衛を担う。普段は燃費を削るだけのコストだが、安全圏を確保するための生命線。緊急時（Effective=2）は逆噴射装置（スラストリバーサー）として機能し、エンジンの排気を斜め前方に跳ね返して先物の強力な慣性（下落エネルギー）を物理的に押し戻し墜落を阻止する。メインエンジン層および姿勢制御エンジン層に装備される。ブースター層への装着不可（1-3参照）。

### **6-3. 随時タスク**

#### **6-3-1. PB利益確定**

監視指標：NQ系→VXN終値。GC系→GVZ終値。IVピーク値＝直近15営業日の最大値（当日除く）。

パターンA（デルタレンジ縮小）：全ポジションを🟡決済承認フローへ

* 現在のデルタ ≤ 初期構築時の50%  
* 含み益 ≥ \+20%

パターンB（ボラ正常化）：50〜100%を🟡決済承認フローへ

* 現在の指数 ≤ IVピーク値×0.85  
* 含み益 ≥ \+40%  
* 残存DTE ≥ 7日

執行禁止条件：Effective=2継続中 OR 含み益 \< \+20%

#### **6-3-2. CCパージ**

以下のいずれかのトリガー発動時に実行する。

**トリガー**

* 高度変更確定時（CCを廃止する方向への移行）  
* Effective=2発動時  
* Emergencyプロトコル発動時

**処理シーケンス**

1. 未約定CC注文を全キャンセル  
2. 既存ポジションをDelta降順（高い順）に決済  
   * Delta ≥ 0.10 → Adaptive Algo Urgent  
   * Delta \< 0.10 → Adaptive Algo Patient  
   * ITM → 即時クローズ（先物含み益との相殺を考慮）  
3. ロールアップ禁止：パージ実行後はトリガー解除まで CC再エントリー禁止

#### **6-3-3. 現金バッファ管理**

* 目標残高：$150k  
* $30k割れ時：T-Bill一部売却提案→承認後執行  
* 現金$100k以上を日次確認
